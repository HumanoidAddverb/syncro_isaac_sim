Data Collection

How to capture teleoperation datasets in NVIDIA Isaac Sim and on the Addverb Syncro5 hardware, the dataset schema, and the SimDataLogger API.

Overview

The pipeline produces datasets in a Parquet + JPEG format compatible with NVIDIA GR00T N1.6 fine-tuning. A unified logging architecture ensures that both simulation and hardware captures use the exact same schema, facilitating co-training strategies where sim data is augmented and mixed with real grounding data.

Step-by-Step

Data collection in NVIDIA Isaac Sim requires two terminals:

Terminal 1 — Launch the Isaac Sim server with logging:

python server_data_logger.py --robot syncro5

Starts the NVIDIA Isaac Sim environment, WebSocket API, and SimDataLogger (30 Hz capture).

Terminal 2 — Connect the teleop client:

python client_with_gripper.py
⚠️ Important
The logger starts recording on launch. Close the sim gracefully to trigger finalize() which writes the Parquet files. Killing the process will lose the dataset.

What Gets Logged

Every 4th physics step (~30 Hz), the NVIDIA Isaac Sim server captures:

  • Joint state — positions of the first 6 joints + gripper (7-dim vector)
  • Action — commanded joint targets (same layout)
  • Ego camera — wrist TiledCamera NVIDIA RGB frame (resized to 320×320)
  • External camera — top-down TiledCamera NVIDIA RGB frame (resized to 320×320)
  • Timestamp — time.time() in seconds
# From server_data_logger.py — logging block
if step % 4 == 0:
    ego_rgb = wrist_cam.data.output["rgb"][0].cpu().numpy()
    ext_rgb = top_cam.data.output["rgb"][0].cpu().numpy()
    joint_pos = robot.data.joint_pos[0].cpu().numpy()

    obs_joints = np.concatenate([joint_pos[:6], [0.0]])  # 7-dim
    action_joints = obs_joints.copy()

    sim_logger.push(action_joints, obs_joints, {
        "rgb_ego": ego_rgb,
        "rgb_external": ext_rgb,
    }, time.time())

SimDataLogger API

The SimDataLogger class in sim_data_logger.py handles background logging.

Initialization

from sim_data_logger import SimDataLogger

logger = SimDataLogger(
    dataset_root="/path/to/output",
    episode_num=0,
    cameras={"ego": {"width": 640, "height": 480}, "external": {"width": 640, "height": 480}},
    fps=30.0,
    dataset_config_path="data_logger_config.yaml",
)

Lifecycle

Method Purpose
init_episode() Create output directories, start background thread
push(action, obs, visual_obs, ts) Enqueue one data frame (non-blocking)
finalize() Stop thread, flush to Parquet, update meta/info.json
discard() Abandon episode, delete written JPEGs

push() Parameters

Parameter Type Shape Description
action_joints np.ndarray (7,) Commanded joint targets
obs_joints np.ndarray (7,) Observed joint positions
visual_obs dict — {"rgb_ego": ..., "rgb_external": ...}
leader_ts float — Timestamp in seconds

Dataset Schema

dataset_root/
├── meta/
│   ├── info.json
│   ├── tasks.parquet
│   └── episodes/chunk-000/file-000.parquet
├── data/chunk-000/file-000.parquet
└── images/
    ├── observation.images.ego/episode_000/*.jpg
    └── observation.images.external/episode_000/*.jpg

Parquet Columns

Column Type Shape Description
frame_number int scalar Sequential frame index
timestamp float64 scalar Wall clock (seconds)
state.joint_positions float32 [7] 7 (arm+gripper)
action.robot float32 [7] Commanded joints
base.timestamp_us int64 scalar Microseconds
mode string — "sim_teleop" or "hw_teleop"
🟢 GR00T Compatibility
The schema matches the NVIDIA GR00T training toolkit format so sim and real datasets can be directly mixed for co-training without conversion.

Hardware Data Collection Addverb

For real-world demonstrations, data is collected directly on the hardware using a logic flow that mirrors the SimDataLogger:

  • TCP/IP Server — A proprietary stack runs on the robot which exposes a TCP/IP server.
  • ROS2 Control — Connection via ROS2 to command joint positions using a teleop client.
  • Unified Logging — Captures ego/external camera images along with robot state, actions, and timestamps into the same Parquet + JPEG format used in simulation.

This shared schema allows researchers to independently collect large-scale augmented simulation datasets and smaller-scale real grounding datasets, then mix them for Strategy 2 co-training.

Multi-Episode Strategies

For GR00T training, you need 50–200 demonstrations. Tips:

  • Each run of server_data_logger.py creates a new dataset directory
  • Task prompts are configured in data_logger_config.yaml → task_ranges
  • Use discard() to abandon bad episodes
  • Multiple directories can be merged by concatenating Parquet files
