Syncro5 Sim-to-Real Training Pipeline

The complete operator's guide for training the Syncro5 cobot — capture VLA datasets in Isaac Sim, fine-tune GR00T models, evaluate policies, and deploy to hardware.

The Five-Stage Pipeline

Stage 1
Data Capture
Log joint states, actions, and dual-camera images via Isaac Sim and physical hardware (TCP/IP stack) → Parquet + JPEG
Stage 2
Train
GR00T N1.6 fine-tune, 50–200 demos, 10k steps
Stage 3
Eval (Sim)
Same Isaac Lab env + DR, ≥50–70% over ≥30 rollouts
Stage 4
Eval (Real)
Policy server + physical Syncro5 arm
Stage 5
Co-Train
Mix real grounded demonstrations with large-scale augmented simulation data to bridge the gap

Technology Stack

The pipeline leverages the NVIDIA robotics ecosystem across every stage — from physics simulation to vision-language-action model deployment.

Isaac Sim
High-fidelity PhysX simulation for the Syncro5 digital twin. GPU-accelerated rendering with domain randomization at 120 Hz.
Isaac Lab
Python-first robot learning framework. Configclasses for scene setup, articulated robots, tiled cameras, and RL/IL environments.
GR00T N1.6
3B-parameter vision-language-action foundation model. Fine-tuned on 50–200 teleoperation demos for autonomous manipulation.
PhysX 5
Real-time rigid body and articulation dynamics. Handles gripper contacts, mimic joints, and collision at simulation-grade fidelity.
USD / OpenUSD
Universal Scene Description format for robot assets, materials, and scene composition.
TiledCamera
GPU-accelerated camera rendering for wrist and top-down viewpoints. RGB + depth at 640×480 @ 30 FPS.

What You'll Learn

  • Set up the Isaac Sim digital twin via syncro_sim_server.py with two cameras mimicking the real world setup.
  • Control the robot and gripper in sim using keyboard teleop via syncro_sim_client.py.
  • Connect to the hardware's proprietary TCP/IP stack via ROS2 to command joint positions.
  • Log synchronized robot state, actions, timestamps, and ego/external camera images for both routes.
  • Collect datasets in the Parquet + JPEG format compatible with GR00T.
  • Fine-tune GR00T N1.6 vision-language-action models for manipulation tasks.

Quick Start

1. Launch the Simulation Server

# Syncro5 with gripper + data logging
python server_data_logger.py --robot syncro5

# Basic simulation (no logging)
python syncro_sim_gripper_revolute.py --robot syncro5

# Custom host and port
python server_data_logger.py --host 0.0.0.0 --port 8765

2. Connect a Teleop Client

python client_with_gripper.py
💡 Tip
Keys q w e r t y increment joints 1–6; a s d f g h decrement them. Press u to open the gripper, j to close it.

3. Use the Python Client Library

from syncro_sim_client import SyncroSimClient

with SyncroSimClient(robot="syncro5") as client:
    client.set_joint_positions({
        "joint1": 0.0,
        "joint2": -0.785,
        "joint3":  1.571,
    })
    state = client.get_state()
    print(state["joint_positions"])

Conventions

  • Joint vector is always [joint1..joint6] (6 arm joints, radians) + optional gripper joints for the Robotiq 2F-85
  • Dataset format is identical between sim and real captures (Parquet + per-episode JPEGs), enabling co-training
  • WebSocket protocol runs on port 8765 by default with JSON messages
